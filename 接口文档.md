# `/query_agents_stream` 接口文档

## 概述

`/query_agents_stream` 是一个**流式接口**，使用 Server-Sent Events (SSE) 协议，允许客户端实时接收数据分析过程中的进度更新和结果。

与同步接口不同，流式接口会在分析过程中持续推送消息，前端可以实时展示处理进度，提升用户体验。

---

## 请求规范

### 请求方法
`POST`

### 请求 URL
```
http://localhost:8011/query_agents_stream
```

### 请求头
```
Content-Type: multipart/form-data
```

### 请求参数

使用 `multipart/form-data` 格式，包含两部分：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `file` | File | 是 | CSV 数据文件 |
| `request_data` | String | 是 | JSON 字符串格式的请求参数 |

### `request_data` JSON 结构

```json
{
  "query": "string",
  "scenario": "string | null"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | 是 | 用户的分析需求描述，如"对销售数据进行聚类分析" |
| `scenario` | string \| null | 否 | 指定分析场景（留空则自动路由）。可选值：`clustering`、`anomaly`、`trend`、`forecast`、`classification`、`association` |

---

## 响应规范

### 响应类型
```
Content-Type: text/event-stream
```

### SSE 消息格式

每条消息的格式为：
```
data: {"type": "事件类型", "data": "事件数据"}

```

### 事件类型

| 事件类型 | 触发时机 | 数据格式 | 说明 |
|----------|----------|----------|------|
| `log` | 分析过程中的任何步骤 | `string` | 进度日志，如"正在初始化计算环境..."、"正在上传数据..." |
| `viz_data` | 可视化数据生成完成 | `array` | ECharts 配置数组（实时推送） |
| `summary` | 分析总结生成完成 | `string` | 分析结论的文本描述 |
| `done` | 整个工作流完成 | `object` | 最终完整结果 |
| `error` | 发生错误 | `string` | 错误信息 |

---

## 事件详解

### 1. log 事件

**示例：**
```json
data: {"type": "log", "data": "正在初始化计算环境..."}

data: {"type": "log", "data": "正在上传数据..."}

data: {"type": "log", "data": "开始执行多智能体工作流..."}
```

### 2. viz_data 事件

当图表数据生成时会立即推送，前端可选择实时渲染：

```json
data: {"type": "viz_data", "data": [
  {
    "type": "scatter",
    "data": {
      "title": "PCA 聚类散点图",
      "series": [...]
    }
  },
  {
    "type": "bar",
    "data": {
      "title": "聚类数量分布",
      "categories": ["分组 1", "分组 2"],
      "series": [...]
    }
  }
]}
```

### 3. summary 事件

分析结论文本：

```json
data: {"type": "summary", "data": "通过 K-Means 聚类分析，将数据分为 3 个簇。轮廓系数为 0.65，表明聚类效果良好。分组 1 占比最大（45%），主要特征为..."}
```

### 4. done 事件

最终完整结果，包含所有数据：

```json
data: {"type": "done", "data": {
  "success": true,
  "message": "通过 K-Means 聚类分析，将数据分为 3 个簇...",
  "echarts": [
    {
      "type": "scatter",
      "data": {...}
    },
    {
      "type": "radar",
      "data": {...}
    }
  ]
}}
```

### 5. error 事件

发生错误时：

```json
data: {"type": "error", "data": "Sandbox not initialized"}

```

## cURL 调用示例

```bash
curl -N -X POST "http://localhost:8011/query_agents_stream" \
  -F "file=@/path/to/your/data.csv" \
  -F 'request_data={"query": "对客户数据进行聚类分析", "scenario": null}'
```

参数说明：
- `-N`: 禁用缓冲，实时显示流式响应
- `-F "file=..."`: 上传 CSV 文件
- `-F 'request_data=...'`: JSON 格式的请求参数

## JavaScript 调用示例

### 使用 EventSource

注意：由于 SSE 不直接支持 multipart/form-data，建议使用 `fetch` 配合 `ReadableStream`：

```javascript
async function streamAnalysis(csvFile, query) {
  const formData = new FormData();
  formData.append('file', csvFile);
  formData.append('request_data', JSON.stringify({
    query: query,
    scenario: null  // 留空自动路由
  }));

  const response = await fetch('http://localhost:8011/query_agents_stream', {
    method: 'POST',
    body: formData
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n');

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const jsonStr = line.slice(6);
        if (!jsonStr.trim()) continue;

        try {
          const event = JSON.parse(jsonStr);
          handleSSEEvent(event);
        } catch (e) {
          console.error('Parse error:', e);
        }
      }
    }
  }
}

function handleSSEEvent(event) {
  switch (event.type) {
    case 'log':
      console.log('[进度]', event.data);
      updateProgressLog(event.data);
      break;
    case 'viz_data':
      console.log('[图表]', event.data);
      renderCharts(event.data);
      break;
    case 'summary':
      console.log('[总结]', event.data);
      displaySummary(event.data);
      break;
    case 'done':
      console.log('[完成]', event.data);
      handleFinalResult(event.data);
      break;
    case 'error':
      console.error('[错误]', event.data);
      showError(event.data);
      break;
  }
}

// 使用示例
const csvFile = document.getElementById('fileInput').files[0];
streamAnalysis(csvFile, '帮我分析这些数据的聚类情况');
```

## 分析场景说明

### 自动路由（推荐）

当 `scenario` 设为 `null` 或不传时，系统会根据查询内容和数据特征自动选择最合适的分析方法。

### 手动指定场景

| 场景值 | 说明 | 适用场景 |
|--------|------|----------|
| `clustering` | 聚类分析 | 客户分群、用户画像、数据分组 |
| `anomaly` | 异常检测 | 欺诈检测、异常交易识别 |
| `trend` | 趋势分解 | 销售趋势分析、时间序列模式识别 |
| `forecast` | 时序预测 | 未来销量预测、数据趋势预测 |
| `classification` | 分类分析 | 类别预测、标签分类 |
| `association` | 关联分析 | 购物篮分析、关联规则挖掘 |

## 常见问题

### Q: 为什么使用 SSE 而不是 WebSocket？

A: SSE 是单向服务端推送，更适合这类"服务端持续推送进度"的场景。实现更简单，原生支持断线重连。

### Q: 如何判断分析完成？

A: 监听 `done` 事件，收到该事件表示整个工作流已完成。`done` 事件包含最终的完整结果。

### Q: 前端需要同时处理 viz_data 和 done 中的 echarts 吗？

A: 不需要。`viz_data` 用于实时展示（可选），`done` 事件包含相同的完整数据。大多数应用只需监听 `done` 事件即可。

### Q: 连接断开怎么办？

A: SSE 标准支持自动重连。如果需要断点续传，建议使用同步接口 `/query_agents`。

## 完整流程时序

```
客户端                              服务端
  |                                   |
  | -------- POST 请求 -------------> |
  |      (file + request_data)        |
  |                                   |
  | <--- "log: 正在初始化..." -------- |
  |                                   |
  | <--- "log: 正在上传数据..." ------ |
  |                                   |
  | <--- "log: 开始执行工作流..." ---- |
  |                                   |
  |      [Router Agent 路由]          |
  |      [Modeling Agent 建模]        |
  |                                   |
  | <--- "log: 正在生成配置..." ------ |
  |                                   |
  |      [Viz Agent 生成图表]         |
  |                                   |
  | <--- "viz_data: [...]" ---------- |
  | <--- "log: 图表数据已生成完毕" --- |
  |                                   |
  |      [Summary Agent 生成总结]     |
  |                                   |
  | <--- "summary: 分析结论..." ------ |
  |                                   |
  | <--- "done: {完整结果}" ---------- |
  |                                   |
```
